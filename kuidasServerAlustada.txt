HOW TO SET UP THE BUILDINGS API SERVER
=======================================

1. NAVIGATE TO PROJECT ROOT

2. CREATE VIRTUAL ENVIRONMENT (first time only)
    python -m venv .venv

3. ACTIVATE VIRTUAL ENVIRONMENT
Mac/Linux:
    source .venv/bin/activate

Windows:
    .venv\Scripts\activate

You should see (.venv) appear at the start of your terminal line.

4. INSTALL DEPENDENCIES (first time only)
    pip install -r requirements.txt

5. SET UP .env FILE (first time only)
Make sure a .env file exists in the project root with:

    DATABASE_URL=postgresql://user:pass@localhost/buildings_db

Replace user, pass, and buildings_db with your actual credentials.

6. START THE SERVER
    uvicorn app.main:app --reload

The --reload flag restarts the server automatically when you edit a file.

7. VERIFY IT'S RUNNING
Open your browser and go to:

    http://localhost:8000/docs

You should see the interactive API documentation.

STOPPING THE SERVER
Press Ctrl+C in the terminal.


COMMON ERRORS
--------------
"command not found: uvicorn"
    → Your virtual environment is not activated. Run step 3 again.

"could not connect to server" (database error)
    → PostgreSQL is not running. Start it with:
       Mac:   brew services start postgresql
       Linux: sudo service postgresql start

"No such file or directory: pildid"
    → Make sure the pildid folder exists in the project root.
       Create it with: mkdir pildid
